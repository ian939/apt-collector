# listing-analyzer 에이전트 설계 문서

> 이 파일은 개발 참조용 문서입니다. 실제 판별 로직은 `src/analyzer.py`에 구현됨.

## 역할

필터를 통과한 아파트 매물에 대해 다음을 판별:
1. **초품아 여부** (초등학교를 품은 아파트)
2. **다주택자 의심 여부** (설명에 명시된 경우만)

## 입력

```json
[
  {
    "id": "매물고유ID",
    "단지명": "레미안 대치팰리스",
    "태그": "초품아, 역세권, 학군지",
    "설명": "초등학교 도보 3분, 급매합니다"
  }
]
```

## 출력

```json
[
  {
    "id": "매물고유ID",
    "초품아": true,
    "다주택자_의심": false,
    "판별_사유": "설명에 '초등학교 도보 3분' 표현 포함"
  }
]
```

## 초품아 판별 기준 (false negative 완화)

조금이라도 근거가 있으면 True로 판별합니다:
- 직접 표현: "초품아", "초등학교 품은", "초품"
- 거리 표현: "초등학교 도보 N분", "초등학교 X분 거리"
- 인접 표현: "OO초 인근", "OO초등학교 담장", "OO초 도보"
- 직접 연결: "창문에서 초등학교", "단지 내 초등학교", "초등 바로 옆"
- 태그: 단지 태그에 '초품아', '초등학교' 등 포함

## 다주택자 의심 판별 기준

매물 설명에 아래 표현이 명시된 경우만 True:
- "다주택자 매물", "다주택 매물", "다주택자매물"
단순 추측이나 여러 채 보유 가능성은 False.

## 프롬프트 언어

한국어. 한국 부동산 도메인 특화 표현 이해에 유리.

## 배치 처리

- 기본 배치 크기: 15건 (10~20건 사이)
- 배치가 너무 크면 중간 매물 판별 품질 저하 위험
- 파싱 실패 시 해당 배치 전체를 "불확실"로 처리 후 계속 진행
